### ชั่วโมงที่ 2: Signal Generation - สร้างสัญญาณด้วยมือเรา (60 นาที)
* **[15 นาที] Concept - Sampling Rate ($f_s$):** อธิบายด้วยภาพว่าเสียงในโลกจริงมีความต่อเนื่อง (Analog) แต่คอมพิวเตอร์เก็บข้อมูลเป็นจุดๆ (Discrete) อย่างไร ยกตัวอย่าง $f_s = 44100$ Hz ในแผ่น CD
* **[30 นาที] Hands-on - Sine & Square Waves:**
    * ให้นักศึกษาทดลองรันโค้ดสร้าง Sine wave 1 วินาที ที่ความถี่ 440 Hz (โน้ต A4)
    * สมการพื้นฐาน: $y(t) = A \sin(2 \pi f t)$ -> แปลงเป็นโค้ด Python: `y = A * np.sin(2 * np.pi * f * t)`
    * ให้นักศึกษาปรับค่าความถี่ (f) และแอมพลิจูด (A) แล้วสังเกตกราฟที่เปลี่ยนไป
* **[15 นาที] Activity - Let's Hear It!:**
    * แนะนำไลบรารี `IPython.display.Audio`
    * ให้นักศึกษาสร้างสัญญาณ 2 ความถี่แล้วนำมาบวกกัน (เช่น สร้างคอร์ดดนตรีง่ายๆ) แล้วกดฟังเสียง
    * **💡 Teaching Tip:** จุดนี้จะเป็น "Aha! Moment" แรกที่นักศึกษาได้เห็นว่าโค้ดที่เขียนสามารถกลายเป็นเสียงได้จริง

    * เปรียบเทียบกราฟระหว่าง "สัญญาณสะอาด" กับ "สัญญาณที่มี Noise"
    * ทดลองฟังเสียงที่ปน Noise

## 🚨 จุดที่ต้องระวัง (Troubleshooting & Common Pitfalls)
1. **ลืมรันเซลล์ก่อนหน้า:** นักศึกษามักจะรันเซลล์ข้าม ทำให้เกิด Error `NameError: name 'np' is not defined` แนะนำให้สอนคีย์ลัด `Runtime > Run before`
2. **Dimension Mismatch:** ตอนรวมสัญญาณ 2 ตัวเข้าด้วยกัน นักศึกษาอาจสร้าง Array ของเวลา (t) ความยาวไม่เท่ากัน ต้องย้ำเรื่องการใช้ `np.linspace` หรือ `np.arange` ให้ขนาดเท่ากันเสมอ
3. **ลืม Import Library:** ย้ำให้เคลียร์และรัน Import block ที่อยู่บนสุดของไฟล์เสมอ
