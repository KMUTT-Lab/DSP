# 🐙 GitHub 101: คู่มือสร้าง Repository แรกของคุณ (The First Base)

ยินดีต้อนรับสู่โลกของนักพัฒนา! **GitHub** เปรียบเสมือนแฟ้มสะสมผลงาน (Portfolio) และคลังแสงสำหรับเก็บโค้ดของคุณ ในวิชานี้เราจะใช้ GitHub เป็น "ฐานทัพ" ในการเก็บและส่ง Lab ทั้งหมด

หากคุณยังไม่เคยใช้งานมาก่อนเลย ให้ทำตาม 2 ขั้นตอนด้านล่างนี้เพื่อเตรียมฐานทัพให้พร้อม:

---

## 🚩 Phase 1: สมัครบัญชีทหารใหม่ (Sign Up)
*(หากคุณมีบัญชี GitHub อยู่แล้ว ให้ข้ามขั้นตอนนี้ไปได้เลย)*

1. เข้าไปที่เว็บไซต์ **[github.com](https://github.com/)**
2. คลิกปุ่ม **`Sign up`** ที่มุมขวาบน
3. กรอกอีเมล (แนะนำให้ใช้อีเมลมหาวิทยาลัยหรืออีเมลหลักที่ใช้ประจำ)
4. ตั้งรหัสผ่าน (Password) และ ชื่อผู้ใช้ (Username) 
   * *คำแนะนำ: ควรตั้ง Username ให้ดูเป็นมืออาชีพและจำง่าย เพราะมันจะกลายเป็น URL ประจำตัวของคุณตลอดไป*
5. ทำตามขั้นตอนยืนยันตัวตนผ่านอีเมลให้เรียบร้อย แล้ว Log in เข้าสู่ระบบ

---

## 🏗️ Phase 2: สร้างโกดังเก็บงาน (Create a Repository)
Repository (เรียกสั้นๆ ว่า Repo) คือ "โฟลเดอร์" บนระบบคลาวด์ของ GitHub ที่เราจะเอาไว้เก็บไฟล์ Google Colab ของวิชานี้

1. เมื่อ Log in เข้ามาแล้ว ให้สังเกตที่มุมซ้ายบน หรือมุมขวาบน แล้วคลิกปุ่ม **`New`** (ปุ่มสีเขียว) หรือคลิกไอคอน **`+`** แล้วเลือก **`New repository`**
2. ในหน้า **Create a new repository** ให้ตั้งค่าดังนี้:
   * **Repository name:** ตั้งชื่อให้สื่อความหมาย เช่น `DSP-Labs` หรือ `Python-DSP-2026`
   * **Description (optional):** พิมพ์อธิบายสั้นๆ (เช่น *พื้นที่เก็บงาน Lab วิชานี้*) หรือปล่อยว่างไว้ก็ได้
   * **Public / Private:** 🚨 **สำคัญมาก!** ให้เลือกเป็น **`Public`** (เพื่อให้ระบบ Colab สามารถดึงไฟล์ไปแสดงผลได้ และเพื่อให้เข้าไปตรวจงานได้)
   * **Initialize this repository with:** ติ๊กถูกที่ช่อง **`Add a README file`** (เพื่อให้ Repo ของเรามีไฟล์เริ่มต้น ไม่ว่างเปล่าจนเกินไป)
3. ตรวจสอบความถูกต้อง แล้วเลื่อนลงมากดปุ่ม **`Create repository`** (สีเขียว) ด้านล่างสุด

---

## 🎉 เสร็จสิ้นภารกิจ!
ตอนนี้คุณมีฐานทัพ (Repository) เป็นของตัวเองแล้ว! URL ที่อยู่บน Address bar ของคุณตอนนี้ (เช่น `https://github.com/ชื่อของคุณ/DSP-Labs`) คือที่อยู่ของคลังเก็บงานนี้ 

เมื่อคุณพร้อมทำ Lab ใน Google Colab และต้องการส่งงาน ให้กลับไปอ่านคู่มือ **`SOP: ขั้นตอนการส่งงาน DSP Lab`** เพื่อดูวิธีบันทึกไฟล์ (Save a copy in GitHub) ลงใน Repository ที่คุณเพิ่งสร้างนี้ได้เลย!
