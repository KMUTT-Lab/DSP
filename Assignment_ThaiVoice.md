
# 🚀 DSP Special Assignment: Project "Thai Voice Localizer"

**สถานการณ์:** หลังจากที่คุณประสบความสำเร็จในการสร้าง AI ที่ฟังภาษาอังกฤษออกใน Lab 03 แต่ในโลกความเป็นจริง เราต้องการเทคโนโลยีที่ตอบโจทย์ผู้ใช้ในท้องถิ่น ภารกิจของคุณคือการ **"สอนภาษาไทย"** ให้กับสมองกล CNN ตัวเดิม เพื่อสร้างระบบควบคุมด้วยเสียงภาษาไทย (Thai Voice Command)

---

## 🎯 วัตถุประสงค์ (Mission Goals)
1. ฝึกฝนทักษะการสร้างชุดข้อมูลเสียง (Audio Dataset Generation) ด้วยตนเอง
2. ประยุกต์ใช้ความรู้เรื่อง **Data Consistency** (16kHz, 1sec, Mono) ในการจัดการไฟล์เสียงจำนวนมาก
3. เรียนรู้การทำ **Model Fine-tuning** หรือการเปลี่ยน Dataset ให้กับสถาปัตยกรรม CNN เดิม
4. วิเคราะห์ผลการทดสอบ (Evaluation) และแก้ไขปัญหาที่เกิดจากความคล้ายคลึงของเสียงในภาษาไทย

---

## 📦 สิ่งที่ต้องเตรียม (The Requirements)
- **สมาชิกกลุ่ม:** 3-5 คน
- **คำศัพท์เป้าหมาย:** เลือกคำศัพท์ภาษาไทย **4 คำ** ที่มีลักษณะเสียงต่างกันชัดเจน (เช่น เดิน, หยุด, ซ้าย, ขวา)
- **จำนวนข้อมูล:** อย่างน้อย **50 ไฟล์ต่อคำศัพท์** (รวมขั้นต่ำ 200 ไฟล์ต่อกลุ่ม) 
    * *Tip: ยิ่งข้อมูลหลากหลาย (หลายคนช่วยกันอัด) AI จะยิ่งฉลาด!*

---

## 🛠️ ขั้นตอนปฏิบัติภารกิจ (Step-by-Step)

### Phase 1: การเก็บรวบรวมข้อมูล (Data Engineering)
1. ใช้เครื่องมือ **[Thai_Voice_Data_Collector.ipynb]** ที่อาจารย์เตรียมไว้ให้ในการอัดเสียง
2. สมาชิกทุกคนต้องช่วยกันอัดเสียง เพื่อให้ AI รู้จักเสียงที่หลากหลาย (Gender, Pitch, Tone)
3. ตรวจสอบให้มั่นใจว่าไฟล์ทั้งหมดถูกตัดขอบและเติมความเงียบจนยาว **1 วินาที (16,000 samples)** เรียบร้อยแล้ว
4. รวบรวมไฟล์จากทุกคนในกลุ่มมาไว้ในโครงสร้างโฟลเดอร์ดังนี้:
   ```text
   /my_thai_dataset
       /word_1 (เช่น เดิน)
       /word_2 (เช่น หยุด)
       ...
   ```

### Phase 2: การฝึกสอนสมองกล (AI Retraining)
1. นำไฟล์ Notebook จาก **Lab 03** มาสร้างสำเนาใหม่
2. เปลี่ยนจุดโหลดข้อมูล (Mission 1) จากการโหลดจาก Google Server ให้เป็นการอัปโหลดโฟลเดอร์ `my_thai_dataset` ของกลุ่มขึ้นไปแทน
3. ปรับค่าพารามิเตอร์ใน **Mission 3** (Output Layer) ให้ตรงกับจำนวนคำศัพท์ภาษาไทยที่คุณเลือก
4. รันการฝึกสอน (Training) และสังเกตกราฟ Accuracy

### Phase 3: การทดสอบและรายงานผล (The Ultimate Test)
1. ทดสอบพูดภาษาไทยสดๆ ผ่านไมโครโฟน (Mission 5)
2. บันทึกผลว่า AI ทายถูกหรือไม่ และมี "คำคู่ไหน" ที่ AI มักจะสับสนกัน
3. **Bonus:** หาก AI ทายผิดบ่อย ลองวิเคราะห์ Spectrogram ของคำนั้นๆ ว่าเกิดจากอะไร? (เช่น เสียงรบกวนเยอะ หรือคำสั้นเกินไป)
